# Редполитика

Задача скилла — помочь проекту зафиксировать голос, словарь и опоры на факты. Редполитика нужна, чтобы следующий текст начинался с контекста: агент сразу знает читателя, площадки, регистр и стоп-слова. Хорошая редполитика короткая, практичная и привязана к живым примерам.

Метод основан на практиках Людмилы Сарычевой ([gladlax.ru](https://gladlax.ru)) и собран совместно с Никитой Архиповым ([niar42.com](https://niar42.com)).

## Шаг 0. Контекст

Проверь, есть ли в проекте файл `.agents/redpolitika.md`.

- Если файла нет — создай его после короткого интервью.
- Если файл есть — прочитай его и обнови только нужные разделы.
- Если пользователь дал примеры текстов — извлеки признаки голоса: длину предложений, обращение к читателю, юмор, степень разговорности, типичные слова.

Интервью держи коротким: 5–7 вопросов. Остальное восстанавливай из примеров и задачи пользователя. Если пользователь не из редакторского мира, задавай вопросы бытовым языком: не «какой регистр по умолчанию», а «тексты должны звучать официально или по-дружески?».

## Режимы

- **Создание (create)** — собрать базовый файл `.agents/redpolitika.md`.
- **Обновление (update)** — добавить новое правило, площадку, слово или пример в существующую редполитику.
- **Диагностика (audit)** — проверить, хватает ли текущей редполитики для работы других скиллов.

Если режим не назван, выбери по запросу: «настрой» — создание, «добавь» — обновление, «проверь» — диагностика.

## Процесс

Подробные вопросы и объяснения — в [references/interview.md](references/interview.md). Итоговый файл собирай по шаблону [assets/redpolitika-template.md](assets/redpolitika-template.md).

### 1. Проект и читатель
Спроси, что за проект и кто читатель. Попроси один абзац, чтобы не перегружать пользователя анкетой.

Правило **живой читатель**: редполитика должна описывать человека и его ситуацию. «Все пользователи» слишком широко; лучше «владельцы кафе, которые сами ведут учёт».

### 2. Площадки
Уточни, где живут тексты: блог, СМИ, соцсети, рассылка, интерфейс, поддержка, документация.

Правило **площадка задаёт регистр**: одна мысль звучит по-разному в статье, пуше и письме. Площадка влияет на длину, прямоту, обращение и степень разговорности.

### 3. Регистр
Выбери регистр по умолчанию:

- **Строгий** — третье лицо, без повелительного наклонения, факты вперёд, сдержанные формулировки. Подходит СМИ, справкам, документации.
- **Авторский** — первое лицо, живой разговор, аккуратный юмор, допустимые разговорные слова. Подходит блогам, рассылкам и личным колонкам.

Если проекту нужны оба регистра, запиши, где какой используется.

### 4. Словарь
Зафиксируй запрещённые и принятые слова: как называем продукт, роли, тарифы, аудиторию, действия.

Правило **словарь проекта главнее вкуса**: если продукт официально называет раздел «кабинет», агент сохраняет это слово. Базовый список стоп-слов бери из `redaktura/references/slovo.md`, затем дополняй словами пользователя.

### 5. Фактура
Запиши, на что проект опирается: личный опыт автора, эксперты-практики, исследования, клиентские истории, данные продукта.

Правило **опора на факты**: будущие тексты должны различать факты, наблюдения и предположения. Если опоры нет, следующий скилл задаёт вопросы по фактуре.

### 6. Примеры голоса
Если пользователь дал 2–3 удачных текста, извлеки из них признаки голоса:

- длину предложений и абзацев;
- обращение, юмор, разговорность, любимые и нежелательные слова.

### 7. Итоговый файл
Создай или обнови `.agents/redpolitika.md`. Если директории `.agents/` нет, создай её.

В шапке файла напиши: «Этот файл читают скиллы redaktura, statya, ux-copy перед работой».

### Режим «Создание»

```

## Что зафиксировано

- Площадки:
- Регистр:
- Словарь:
- Фактура:

## Что ещё нужно узнать
[дыры в контексте — вопросы; если их нет, раздел опусти]
```

```

|---|---|---|---|

```

```

## Главные дыры
| Раздел | Что мешает | Что спросить или добавить |
|---|---|---|
```

## Чего не делать

- **Не писать энциклопедию.** Редполитика — рабочая памятка, её должно быть легко прочитать перед правкой.
- **Не выдумывать факты и контекст.** Если неизвестны аудитория, площадки, словарь или фактурные опоры, задай вопрос или пометь дыру в «Что ещё нужно узнать».
- **Не выдумывать голос.** Если примеров нет, честно пометь предположения и спроси пользователя.
- **Не переносить стоп-слова механически.** Базовый список помогает начать, затем решает словарь проекта.
- **Не ломать существующий файл.** При обновлении сохраняй старые правила, если пользователь не просил их заменить.

## Связанные скиллы

- **redaktura** — читает `.agents/redpolitika.md` перед правкой текста
- **statya** — использует редполитику для брифа, регистра и словаря
- **ux-copy** — использует редполитику для интерфейсного тона и терминов продукта
